Movie Magic?

How successful is a movie? Is it the money it makes, or how much audiences enjoy it? Thats what I want to explore. Using data, I hope to uncover patterns that reveal different kinds of success whether financial, critical, or something in between. Rather than predicting outcomes, this project uses clustering, an exploratory technique in data mining that groups similar movies based on shared characteristics. The goal isnt to say what will make a future movie successful, but to see how existing movies naturally group together when we look at their budgets, revenues, and ratings. Through these patterns, I hope to gain insight into the different types of success films can achieve.

The Movies

Banik, R. (n.d.). The Movies Dataset [Data set]. Kaggle. https://www.kaggle.com/datasets/rounakbanik/the-movies-dataset

The dataset titled The Movies Dataset (hosted on Kaggle) contains metadata for over 45,000 movies specifically, films listed in the full MovieLens 20M Dataset collection. It also includes approximately 26 million ratings from more than 270,000 users.

The movies_metadata.csv file includes features like budget, revenue, genres, production companies/countries, original language, release date, etc. The ratings_small.csv or ratings file captures user-movie ratings userId, movieId, rating, timestamp.

The dataset covers movies released on or before July 2017.

Casting Call

Clustering is an unsupervised machine learning technique used to group similar items together based on their characteristics. Unlike supervised learning, clustering does not rely on labeled outcomes. Instead, the algorithm looks for patterns in the data and organizes observations into groupscalled clusterswhere items in the same cluster are more similar to each other than to items in other clusters.

In this project, clustering is useful because we are trying to understand what makes a movie successful without defining success beforehand. We are letting the data reveal natural groupings based on patterns in budget, revenue, and audience ratings.

Before diving into the analysis, I need to understand what data I have to work with. The project will draw primarily from two CSV files in The Movies Dataset: movies_metadata.csv and ratings_small.csv. The movies_metadata file contains descriptive information about each film, such as titles, budgets, revenues, and production details, while the ratings file captures individual user ratings for movies.

Need Subtitles

Before clustering, I need to scale the numeric features. The values for budget and revenue are extremely large compared to vote_average and vote_count. If I feed these raw values into K-Means, the algorithm will focus almost entirely on the largest numbers and ignore the smaller ones.

To prevent this, I apply StandardScaler, which transforms each feature so that it has a mean of 0 and a standard deviation of 1. This puts all variables on the same scale, ensuring that each contributes equally to the clustering process.

In[9]:
X = movies_clean[['budget', 'revenue', 'vote_average', 'vote_count']]

scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

Lights......

To perform K-Means clustering, I need to choose a value for k, which is the number of clusters. However, the dataset doesnt tell me what k should be. To determine this, I use the Elbow Method.

I calculate inertia for different values of k. Inertia is the sum of squared distances between each data point and its assigned cluster center. Lower inertia means tighter clusters. I then plot the inertia against k.

As k increases, inertia decreases but only up to a point. Eventually, the decrease slows down and the plot begins to flatten. The point where the curve makes a noticeable bend, resembling an elbow, indicates a good balance between cluster accuracy and complexity.

This elbow point is the recommended choice for k because:

Smaller k values oversimplify and merge distinct patterns.

Larger k values add complexity without meaningful improvement.

In my case, the elbow appears around k = 4 to 5, so I select one of these values to proceed with clustering.

In[10]:
inertia = []

for k in range(1, 10):
    km = KMeans(n_clusters=k, random_state=42)
    km.fit(X_scaled)
    inertia.append(km.inertia_)

plt.plot(range(1, 10), inertia, marker='o')
plt.xlabel('Number of Clusters (k)')
plt.ylabel('Inertia')
plt.title('Elbow Method')
plt.show()

Camera!

Now that the optimal number of clusters has been chosen, I fit the K-Means model to the standardized feature data and assign each movie to a cluster. This adds a new column, cluster, indicating which group each movie belongs to. I then display the first 10 rows to verify that the clustering was applied correctly.

I chose K-Means because:

-The dataset is large, and K-Means handles large datasets efficiently. Agglomerative Clustering becomes computationally expensive as dataset size grows.

-K-Means produces clear, well-separated clusters when using numeric, scaled data, which fits this situation well.

-K-Means allows direct control over the number of clusters (k) and provides a measurable performance metric (inertia) used in the Elbow Method to justify the choice of k.

The Reviews are In!

The heatmap allows us to compare how each cluster behaves across budget, revenue, audience ratings, and vote counts (audience engagement). A clear pattern emerges: films with higher budgets tend to show higher audience engagement and, in some cases, better average ratings. This makes sense larger budgets often mean wider releases, more marketing, and more visibility, which increases the number of votes.

However, the relationship is not linear. The cluster with the second-highest revenue does not have the second-highest average rating. Instead, the cluster with the third-highest revenue actually shows stronger audience approval.

This suggests that:

Spending more money helps a movie get seen, but does not guarantee audience satisfaction.

There may be a threshold where increasing budget (and therefore revenue potential) no longer improves audience perception possibly because marketing hype can drive people to theaters, but cannot force them to like the movie.

Impact

This analysis suggests that there are budget thresholds where increases in spending lead to higher audience engagement and visibility. In other words, larger budgets tend to correlate with higher revenue and more audience interaction, likely because higher-budget films receive wider releases and stronger marketing campaigns. However, this does not mean that high budget automatically results in a better film it only increases the likelihood that more people will see the film and rate it. The data implies that success follows attention, not necessarily quality.

However, we need to be careful about how this insight is interpreted. If studios rely too heavily on patterns like this, it could reinforce a cycle where only high-budget productions are considered worth making, potentially pushing out smaller or independent films that dont have the same financial backing. This creates a cultural cost: fewer risks, fewer new voices, and fewer stories outside the mainstream.

This project is also limited by the dataset itself. Although the original dataset contained over 45,000 entries, most were unusable because of missing or incomplete financial information. The conclusions drawn here are based on the much smaller subset of ~3,800 films that reported full data. Its also worth noting that the dataset doesnt separate production budget from marketing spend, which is often a major factor in visibility and engagement. Without that, the analysis can only point to correlations, not causes.

In short:

We can say: Higher budgets tend to increase audience engagement and revenue.

We cannot say: Higher budgets make better movies.

The project provides insight into how financial scale influences audience reach, but a complete answer to what makes a film successful would require additional data including marketing, genre, distribution scope, and cultural context.

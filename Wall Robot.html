Wall-E?

Robots are blind. How do they then move around? This dataset contains ultrasonic sensor data for a robot. Can we predict what this robot will do? More importantly, does having more sensors help or hurt? Can we reduce the cost?

Data Set

Freire, A., Veloso, M., & Barreto, G. (2009).
Wall-Following Robot Navigation Data [Dataset]. UCI Machine Learning Repository.
https://doi.org/10.24432/C57C8W.

This data set is that of a robot traveling along a wall and the sensor readings + the actions it took given those readings.
Rows represent one moment in time. It has 24 sonar sensors that measure distances around the robot and the last column is the action of robot. This data set has 5456 rows. Feature set is comprised of these 24 sensor measurements and the target value consists of 4 classes: Move-Forward, Slight-Right-Turn, Sharp-Right-Turn, Slight-Left-Turn.

Visualizing Feature Independence

To assess whether the features are independent, I plotted a correlation heatmap of the sensor readings.
If features were independent, the off-diagonal values in the heatmap would be close to zero (white/blue).
Instead, the heatmap shows several regions of strong positive and negative correlation, indicating that many sensors are redundant or related.

Because Gaussian Naive Bayes assumes conditional independence between features, this high correlation violates the models assumptions and leads to poor accuracy.
Therefore, I pivoted to a Random Forest classifier, which is more robust to correlated features and can exploit complex interactions between them.

Stress testing with cross validation

To ensure the Random Forest model was not just performing well on a lucky split, I applied k-fold cross-validation (k=5). This method trains and tests the model across multiple partitions of the dataset, providing a more reliable measure of generalization.

The results were consistent, with accuracies around 99% across all folds and a very low standard deviation (~0.002). This indicates that the model is not overfitting to a single split but is robust across the dataset.

While the performance appears almost "too good," this is likely due to the curated nature of the dataset (UCI robotics), where the class boundaries are clear and sensor readings are well-structured. In practice, noisier real-world data may yield lower performance, but the cross-validation confirms the model is behaving as expected here.

Comparison

As a further check, I implemented a Gaussian Naive Bayes model since it was my initial choice prior to examining the heat map. This served two purposes: first, to confirm the hypothesis that Naive Bayes would underperform due to strong feature correlations; and second, to verify the validity of the dataset. If Naive Bayes had achieved similarly high performance, it would suggest the dataset might be skewed or overly simple. However, its poor performance compared to Random Forest confirms both the earlier hypothesis and that the dataset meaningfully distinguishes between algorithms.

Less is more

I modeled the dataset using only the 4 forward-facing sensors provided in the UCI dataset. The model still achieved very high accuracy. This suggested that reducing the number of sensors did not significantly degrade performance.

The consistency of accuracy across models raises the possibility that one or more sensors strongly dominate decision-making in this robots behavior. In other words, a single featureor a subsetmay be disproportionately responsible for determining the robots actions. This warrants further investigation into feature importance and redundancy in the dataset.

Visualize Sensors

To understand which inputs drive the classifier, I first examined per-sensor contribution rather than raw reading counts (counts can be misleading as every sensor records once per sample). I used model-based feature importance to identify dominant sensors and then validated those findings with an ablation study.

The results show Sensor 14 (15 forward-facing) dominates. Using Stratified 5-fold CV on subsets of the top-ranked features:

  • Top 1 sensor (Sensor 14) ~70% accuracy

  • Top 23 sensors >90% accuracy

  • Top 48 sensors 9899% (plateau; additional sensors add little)

Interpretation. The classifiers performance depends mainly on a narrow forward-facing arc. This suggests the dataset is highly deterministic and redundant: a few sensors largely determine the label, and the model needs little additional context. Rather than learning a rich policy, the model is effectively capturing a simple trigger behaviorconsistent with a robot that drifts until a forward sensor reports a close obstacle, then over-corrects.

What this adds to the story.

  • Confirms why Naive Bayes underperforms (correlated sensors) and why Random Forest excels (handles correlation and interactions).

  • Shows the robots control signal is concentrated in the forward sensors, raising questions about generalization to different walls/environments and whether peripheral sensing is underutilized.

Impact

The findings have a larger implication: a robot trained on deterministic data like this will perform well in the lab but fail in new environments. That risk is the same in bigger autonomous systems like self-driving cars, where brittle models can mean life-threatening failures. This dataset shows how relying only on forward sensors might be good enough for a narrow case but is dangerous for general deployment. It highlights the need for more robust, diverse data collection and careful validation before trusting models in safety-critical systems.

Story Telling

When I started this project, I expected to use a Gaussian Naive Bayes model, but the data quickly proved otherwise. Random Forest delivered extremely high accuracy, which at first looked like success but really highlighted how deterministic the dataset was. That turned the project into more of an exploration of the dataset itself than just training a classifier. What I learned is that even when accuracy looks great, it doesnt always mean the model is smart sometimes its just memorizing. But we can indeed know what the robot will do and that 24 sensors is overkill and the sweet spot for this particular robot and this particular wall is 4-8 sensors.
